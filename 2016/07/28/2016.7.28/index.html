我的博客搭建过程踩到的坑

发表于 2016-07-28   |   分类于 技术   |     |   阅读次数

写在前面的话


　本篇博文主要记录在本博客创建过程中，本人遇到的大大小小的坑。不是完整的博客创建教程。在本博文的末尾，我将贴出在建博过程中，所用到的自认为借鉴意义比较大的博文。希望能对也想建博的童鞋有所帮助。共勉！

坑一：万网域名的解析

万网的域名申请下来之后，可以在万网的管理控制台解析，也可以在DNSPod里解析，二选一就好。都设置可能会发生解析冲突的问题

下图所示的就是就是云解析在万网中的位置，点击你的域名就好：

进去之后按着下图所示的去设置参数：

　　我划掉的部分就是你Blog在Github中托管的Repository的名字（格式为：username.github.io,每一个Github账户都只能有这么一个名字的文件夹，专门用来管理博客。）
A配置的IP地址是Github自己的IP地址，就是要把你的域名和Github平台连在一起的。到本Blog建成的时候起，这个IP并没有改变。要是，你所有设置都正确，却连不上Github,那你就要考虑是不是这个IP变了，记得去Github官网查查，改了就行。

接下来的是重点:

　　万网的解析时间为0-72个小时，设置好了一定要耐心等待。还有，就是你的域名一定要实名注册，不然万网那边是不会给你解析通过的，就像下图中的那样。我在这里走了很多坑，希望大家引以为戒。

总结：

　　网上有很基于Github和Hexo建站的教程，已经很成熟。然而，在实际做的过程中，还是多多少少各种坑。除了自己第一次动手没经验以外，我觉得更主要的原因是对很多步骤的原理都没有加入自己的思考，就照着别人的教程跟着瞎走。每个人遇到的问题是不一样的，做事情的思路和方法也会有所差别，所以，可能看2、3篇教程走走，自己就走蒙了。后果就是，当遇到问题的时候，自己都很难去定位问题到底是出在哪里的。

　　所以，我的意见是，在做事情之前，要先想清楚大致都要经过哪些步骤，心有沟壑，方能有的放矢。这也是我这次最大的收获吧。

个人建博流程：

本地：Git + Hexo +Node.js

远程：Github帐号+Github仓库+个人域名（可选，Github提供http://username.github.io的个人域名）

　　远程部分的内容比较简单。本地部分的关键在做好config.xml文件的配置。至于更换主题什么的，建议在个人博客做好之后，再一步一步慢慢来。
